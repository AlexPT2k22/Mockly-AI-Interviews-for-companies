# Mockly AI-Powered Interview Practice Platform

A full-stack SaaS application that generates adaptive interview questions, transcribes voice answers in real time, and delivers AI-powered feedback with structured multi-dimensional scoring.

**Live Demo:** [mockly-alpha.vercel.app/showcase](https://mockly-alpha.vercel.app/showcase)

---

## Overview

Mockly helps job seekers practice and improve their interview skills through realistic, AI-driven simulations. Users select an interview focus (frontend, backend, system design, or behavioral), receive a context-aware question generated by GPT-4o, answer via voice or text, and get instant feedback with scores across four dimensions -- clarity, depth, structure, and relevance.

The project was built to demonstrate full-stack engineering capability across a modern JavaScript/TypeScript toolchain: React single-page application, Node.js REST and WebSocket server, PostgreSQL database with Row-Level Security, and integration with four external AI and communication APIs.

The server operates in a graceful mock mode when no API keys are configured, making the entire application fully functional for local development and demo purposes without any external dependencies.

## Architecture

```
[React SPA on Vercel]
        |
        | HTTP + WebSocket
        v
[Express API on Render]
        |
        +---> OpenAI (GPT-4o QA, Whisper STT, TTS)
        +---> Deepgram (real-time streaming STT)
        +---> ElevenLabs (neural TTS voices)
        +---> Supabase (PostgreSQL: waitlist, beta feedback)
        +---> Resend (transactional email)
```

The frontend communicates with the backend through REST endpoints for question generation, answer analysis, batch transcription, and TTS synthesis. A separate WebSocket connection streams audio chunks to Deepgram for real-time speech-to-text during voice recording sessions, with automatic fallback to batch-mode Whisper transcription when streaming is unavailable.

## Key Features

- **Real-time WebSocket streaming transcription** via Deepgram, with automatic fallback to batch-mode Whisper transcription when streaming is unavailable. Audio chunks are sent over a persistent WebSocket connection for near-instant results.
- **Adaptive AI question generation** using GPT-4o-mini across four categories (frontend, backend, system design, behavioral). Questions are context-aware and non-repetitive.
- **Multi-dimensional answer scoring** evaluating clarity, depth, structure, and relevance. Each dimension receives a 0-100 score with a progress bar and natural-language feedback.
- **Voice recording** via the MediaRecorder API with WebM chunking, batch upload to Whisper for transcription, and seamless switching between voice and text input mid-session.
- **Filler word detection** highlighting common fillers (uh, um, like, you know) in transcripts with severity markers and improvement suggestions.
- **TTS narration** of questions and feedback using OpenAI TTS or ElevenLabs neural voices, with audio queuing for sequential playback.
- **Waitlist system** with Zod-validated signup forms, Supabase persistence with unique email constraint, and Resend-powered welcome emails containing discount codes.
- **Beta feedback collection** with star ratings and optional comments, upserted per email to Supabase.
- **Session state persistence** to localStorage, enabling interview state recovery across page reloads.
- **Graceful mock mode** that provides deterministic fallbacks for every AI endpoint when no API keys are configured, allowing full-featured local development.
- **Responsive design** with glass-morphism aesthetics, scroll-triggered animations, and a mobile-first navigation system.

## Getting Started

### Prerequisites

- Node.js >= 18
- npm >= 9

### Frontend

```bash
cd frontend
npm install
npm run dev
```

The development server starts at `http://localhost:5173` with HMR enabled. API calls are proxied to the backend via the Vite config.

### Backend

```bash
cd server
npm install
npm run dev
```

The API server starts at `http://localhost:8787`. Without any API keys configured, the server runs in mock mode with deterministic responses.

### Environment Variables

Create a `server/.env` file. All variables are optional -- the server runs in mock mode when keys are missing.

| Variable | Purpose | Required | Default |
|---|---|---|---|
| `PORT` | Server port | No | `8787` |
| `OPENAI_API_KEY` | OpenAI API key for GPT-4o, Whisper, TTS | No | (mock mode) |
| `AI_MODEL_QA` | Model for question generation and analysis | No | `gpt-4o-mini` |
| `AI_MODEL_TTS` | Model for text-to-speech | No | `gpt-4o-mini-tts` |
| `DEEPGRAM_API_KEY` | Deepgram API key for streaming STT | No | (mock mode) |
| `ELEVENLABS_API_KEY` | ElevenLabs API key for neural TTS voices | No | (mock mode) |
| `ELEVENLABS_VOICE_ID` | ElevenLabs voice ID | No | `21m00Tcm4TlvDq8ikWAM` (Rachel) |
| `RESEND_API_KEY` | Resend API key for transactional email | No | (mock mode) |
| `FROM_EMAIL` | Sender address for emails | No | `noreply@mockly.com` |
| `SUPABASE_URL` | Supabase project URL | No | (mock mode) |
| `SUPABASE_ANON_KEY` | Supabase anonymous key | No | (mock mode) |
| `ALLOWED_ORIGIN` | CORS allowed origins (comma-separated) | No | `http://localhost:5173,https://mockly-alpha.vercel.app` |
| `MAX_AUDIO_BYTES` | Maximum audio upload size in bytes | No | `5_000_000` |

### Mock Mode

When `OPENAI_API_KEY` is not set, every AI endpoint returns deterministic results:

- `/api/ai/question` -- Returns a placeholder question based on category
- `/api/ai/analyze` -- Returns pseudo-metrics and generic feedback
- `/api/ai/transcribe` -- Returns a canned transcription string
- `/api/ai/tts` -- Returns a short silent WAV buffer
- `/api/waitlist` and `/api/beta` -- Store data in memory instead of Supabase

This allows the entire application to function without any external API dependencies, which is ideal for local development and demo purposes.

## API Reference

Base URL: `https://landingpagewaitlistinterview.onrender.com`

### Health

| Method | Path | Description |
|---|---|---|
| GET | `/api/health` | Returns `{ status, mock }` -- indicates if running in mock mode |

### AI Routes

| Method | Path | Body | Description |
|---|---|---|---|
| POST | `/api/ai/question` | `{ category: string }` | Generate an interview question |
| POST | `/api/ai/analyze` | `{ answer: string }` | Analyze answer, return scores and feedback |
| POST | `/api/ai/transcribe` | `FormData(audio: file)` | Transcribe audio file via Whisper |
| POST | `/api/ai/tts` | `{ text: string, provider: string }` | Synthesize speech as WAV |
| POST | `/api/ai/analyze-transcript` | `{ transcript: string }` | Detect filler words, hesitations, speech patterns |

### Waitlist Routes

| Method | Path | Body | Description |
|---|---|---|---|
| POST | `/api/waitlist` | `{ email, firstName, lastName, ... }` | Join waitlist, trigger welcome email |
| GET | `/api/waitlist/count` | -- | Return total waitlist signups |

### Beta Routes

| Method | Path | Body | Description |
|---|---|---|---|
| POST | `/api/beta/feedback` | `{ email, rating, comment }` | Submit or update beta feedback |
| GET | `/api/beta/summary` | -- | Return average rating, total, distribution |

### WebSocket

| Protocol | Path | Description |
|---|---|---|
| WebSocket | `/ws/transcribe` | Real-time streaming transcription via Deepgram |
